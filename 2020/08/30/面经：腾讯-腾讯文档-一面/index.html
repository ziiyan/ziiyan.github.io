面经：腾讯-腾讯文档-一面

2020-08-30 面试 评论

CSS

  • position 取值

    • static 是 position 的默认取值，取 static 的时候 top、right、bottom、left、z-index 无效

    • relative 是相对定位，不改变页面布局，相对于原来的位置偏移

    • absolute 会将元素移出正常文档流，相对于元素最近的非 static 定位的祖先元素偏移

    • fixed 也会将元素移出正常文档流，它是相对于视口偏移的，屏幕滚动时不改变

    • sticky 根据正常文档流定位，相对于最近块级祖先定位，top、right、bottom、left 设定阈值

  • 水平居中、垂直居中（之前整理过又忘了 😫）

    • 水平居中

      首先如果是行内元素，可以设置 text-align: center，块级元素的话把 display 设置为 inline-block 也可以用这种方式，但这种方式文本也会居中，需要特殊处理一下。

      然后如果是块级元素可以设置元素 margin 为 0 auto

      还有就是用弹性布局，在父元素上设置 display: flex、justify-content：center

      还可以利用绝对定位，首先在父元素上设置 position: relative，子元素上设置 position: absolute、left: 50%、transform: translateX(-50%)；如果知道元素宽度可以把 transform 换成 margin-left: 元素宽度 * -0.5

    • 垂直居中

      垂直居中也是分行内元素和块级元素，当子元素的 line-height 等于父元素的高度时，子元素就会相对于父元素垂直居中。

      块级元素可以用弹性布局，父元素设置 display: flex，align-items: center

      绝对定位和水平居中类似，父元素设置 postion: relative，子元素设置 position: absolute、top: 50%、transform: translateY(-50%)，如果知道元素高度可以用 margin-top: 元素高度 * -0.5 代替 transform

    • 水平垂直居中：

      我会首选弹性布局，因为平时也用得比较多。设置父元素 display: flex、justify-content:center、align-items: center，或者父元素只设置 display: flex，子元素设置 margin: auto

      还有一种是利用绝对定位，父元素设置 position: relative，子元素设置 position: absolute、top: 50%、left: 50%，transform: translate(-50%, -50%)，如果知道元素的宽高可以用 margin-top 和 margin-left 来代替 transform，margin-top: 元素高度* -0.5、margin-left: 元素宽度 * -0.5

    • transform 取值

      CSS 变换函数，有 translate、rotate、scale、skew

  • css3 动画和 js 动画的区别

    纯 js 动画一般是通过 setInterval 或者 setTimeout 方法的回调函数来持续调用改变某个元素的 css 样式，这样做通常会导致页面频繁地重排重绘。而且 js 是在浏览器的主线程中运行的，主线程还需要运行其它 js 脚本，会和动画抢占线程的控制权，可能会导致丢帧。这个可以通过使用 requestAnimationFrame 进行优化，js 动画的优点是控制能力很强，相比起来 css3 的控制能力就比较弱，只能通过 animation-play-state 属性来控制开始或者暂停，但 css3 动画是不需要用到主线程的，所以不会存在抢占主线程的情况，也就不会丢帧，性能比较好，但可能会有兼容性的问题。

    • 重排重绘

      浏览器在绘制页面的时候用到了主线程和合成器线程。

      首先主线程解析 html 构造 DOM Tree，解析 CSS 计算节点样式，然后遍历 DOM Tree 和计算好的样式生成 Layout Tree，这里就是布局阶段，然后主线程根据 Layout Tree 生成 Layer Tree 和绘制列表，这里就算是绘制阶段，接着主线程把 Layer Tree 和绘制列表提交给合成器线程，真正的绘制操作是由合成器线程执行的。

      重排就是指改变了样式布局，重新执行 Layout 以及之后的阶段，重绘是重新执行了绘制阶段以及之后的阶段，跳过了布局阶段。所以重排的损耗比重绘大。

      但无论是重排还是重绘，都是发生在主线程的，而css3 的 transform 是直接在合成器线程完成的，不会抢占主线程，所以性能会更好。

      • 导致重排的操作
        • 更改布局，比如位置、尺寸
      • 导致重绘的
        • 不改变布局的，比如颜色、阴影
      • 优化
        • 合并样式操作
        • 隐藏元素（脱离文档流），修改完再显示

    • requestAnimationFrame

      requestAnimationFrame 让页面重绘的频率与浏览器刷新频率保持同步，主线程优先处理requestAnimationFrame 的回调，不至于被其它脚本抢占主线程导致丢帧

  • animation 取值

    • name 动画名称
    • duration 持续时间
    • function 改变速度
    • delay 延迟
    • count 次数 infinite
    • direction 方向
      • normal
      • alternate
      • reverse
      • alternate-reverse
    • fill-mode：CSS动画在执行之前和之后如何将样式应用于其目标
      • forwards
      • backwards
      • both
    • play-state
      • running
      • paused

JS

  • 纯 ES5，不用 ES6 如何实现继承

  • （上一题没答出来）换种方式问，讲讲原型和原型链

    Js 的每个对象都有一个__proto__属性指向构造函数的原型对象，这个原型对象也有自己的原型对象，以此类推直到原型对象为 null。当我们访问一个对象的属性的时候，不仅会访问对象本身，还会沿着__proto__属性访问原型对象，还有原型对象的原型对象……直到找到匹配的属性或者原型链末尾。

  • ES6 用过哪些特性

    • let、const

  • let 和 var 有什么区别

    • map、set、class（没追问）

  • 讲讲箭头函数

    箭头函数没有自己的 this，不能用作构造函数，和 new 一起用会抛出错误。然后我之前在浏览器控制台打印过，箭头函数是有 __proto__ 属性，但没有 prototype 属性的。

  • 箭头函数的 this

    箭头函数的 this 就是定义时所在的对象，而不是调用时所在的对象，所以箭头函数中的 this 是固定的，在定义箭头函数的时候就决定好了。

  • 讲讲 promise

    Promise 是异步编程的一种解决方案，解决了传统回调和事件方式造成的回调地狱的问题。它一共有三种状态，pending，fulfilled，rejected，状态只受异步操作的结果影响，而且只有 pending 到 fulfilled 或 rejected 这两种可能，状态一旦改变就凝固了，不会再发生改变。然后因为 Promise 只有这三种状态嘛，但 pending 这个异步过程是有可能持续一段时间的，我们就没办法知道进度，就是说这个 pending 是刚开始还是快结束这样。而且 Promise 一旦新建了就会马上执行，所以是没有办法中途取消的，如果我们不设置回调函数，比如 catch，内部抛出的错误就无法反应到外部。除此之外，then 返回的是一个新的 Promise，所以支持链式调用。

  • promise 有什么优点和缺点

    首先优点是解决了回调地狱的问题，写法更加自然，这也是 promise 出现的意义。然后它作为一个规范的话可以说是统一了异步编程的写法吧。

    缺点的话，就是无法取消，因为 promise 一旦新建就会立即执行，而且它的状态不受外界影响，只和异步操作的结果有关系。第二点是错误需要用回调函数来捕获，比如 then 的第二个参数，或者直接用 catch。第三点的是 promise 只有三种状态，pending、fulfilled、rejected，然后处于 pending 的时候，其实有可能持续很长时间，这边就没办法知道进度，也就是说这个 pending 是刚开始还是快结束这样的。

    首先先定义一个 Promise 类，然后我们新建的时候是传入一个函数作为参数的，这个被传入的函数有两个参数 resolve 和 reject，用的时候是说 js 引擎提供，手写的话就得内部定义了。然后新建 Promise 的时候是立即执行的，在构造函数里面直接调用传入的参数。

网络

  • HTTP 状态码

    首先状态码可以分成五类，分别是12345开头。

    1 开头的是信息响应，我见过 101，就是开启 websocket 的时候需要切换协议。

    2 开头是一些成功的响应，200 是最常见的成功嘛。

    3 开头是重定向相关的，301 是永久重定向，302 是临时重定向，304 是 Not Modified，就是内容和上次请求对比未修改，可以直接用缓存。

    4 是客户端错误，401 是用户验证出错，403 是服务器理解请求但拒绝执行，比如说服务器通过 session 已经知道用户身份了，但用户没有权限，这种情况前端需要跳转到登录界面。

    5 是服务器错误。

    • 301、302、304、403

      301 是永久重定向，302 是临时重定向，304 是 Not Modified，403 是服务器拒绝执行。

  • HTTPS 为什么安全

    首先 HTTPS 是为了解决 HTTP 的安全问题出现的，之所以说 HTTP 不安全主要有三个方面的原因。一是明文传输，内容可能被窃听；二是无法验证传输内容完整性，内容可能被篡改；三是不验证身份，可能被伪装。

    所以针对这些问题，HTTPS 首先会对内容进行加密。加密的话有两种加密，一种是对称加密，加密和解密用的都是同一个密钥；另一种是非对称加密，就是公钥加密，私钥解密。在 HTTPS 中，数据交换阶段用到的是对称加密，因为性能比非对称加密好一点，然后在数据传输之前，需要通过非对称加密来交换对称加密用到的这个密钥。

    这个过程是服务器把自己的公钥交给 CA，然后 CA 用自己的私钥给服务器的公钥签名之后颁发证书给服务器。客户端发起 HTTPS 请求后，服务器把 CA 颁发的证书返回给客户端，客户端拿到证书后用浏览器内置的 CA 公钥进行验签，就验证了证书的真实性，也得到了服务器的公钥。

    然后客户端生成一个用来对称加密的共享密钥，用服务器的公钥加密，发给服务器，然后服务器用自己的私钥解密，就得到了这个共享密钥。之后就开始数据传输阶段，就可以使用这个双方都有的共享密钥来加解密了。

    数据完整性是用到数字签名，就是发送方用摘要算法对报文提取字段生成一个摘要，使用私钥对摘要进行加密后，作为数字签名附在报文上。接收方收到报文后使用相同的摘要办法再提取一次摘要，然后使用公钥对数字签名进行解密，把解密的结果和自己刚刚算的摘要比对一下，相同就说明没被篡改。
