面试准备：闭包

2020-09-09 JavaScript, 面试 评论

基础

js 代码执行过程有两个阶段：代码编译（编译器） -> 代码执行（引擎）

编译器：

  • 词法分析
  • 语法分析
  • 可执行代码生成
  • 作用域规则确定

js 引擎：

  • 创建执行上下文
  • 代码执行
  • 垃圾回收

执行上下文

  • JavaScript引擎会以栈的方式来处理它们，这个栈，我们称其为函数调用栈(call stack)。栈底永远都是全局上下文，而栈顶就是当前正在执行的上下文。

执行上下文生命周期

  • 创建
    • 生成变量对象
    • 建立作用域链
    • 确定 this 指向
  • 执行
    • 变量赋值
    • 函数引用
    • 执行其他代码
  • 出栈，等待垃圾回收

作用域

  • 一套规则，定义引擎如何在当前作用域以及嵌套的子作用域中根据标识符名称进行变量查找

作用域链

  • 是由当前环境与上层环境的一系列变量对象组成，它保证了当前执行环境对符合访问权限的变量和函数的有序访问

闭包

闭包是一种特殊的对象。

它由两部分组成。执行上下文(代号A)，以及在该执行上下文中创建的函数（代号B）。

当B执行时，如果访问了A中变量对象中的值，那么闭包就会产生。

通过闭包，我们可以在其他的执行上下文中，访问到函数的内部变量

什么是闭包

（自己描述的）JavaScript 中有一个执行上下文，闭包就是把函数和它的在上下文中的引用捆绑在一起形成的。

比如说我们定义了一个函数 a，然后在函数 a 里面定义了一个函数 b，那么这个 b 所处的执行上下文就是函数 a 的这个函数作用域，如果 a 中还有一个变量 c，函数 b 里面是可以引用这个变量 c 的，在函数 b 里面访问到外部函数作用域 a 中的变量 c，这就形成了闭包。

我们可以把这个函数 b 暴露给全局对象，比如赋值给一个全局变量 w，那么我们可以通过 w 访问到函数 a 的内部变量 c。

应用场景

  • 柯里化
  • 模块化
