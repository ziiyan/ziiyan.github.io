  • symbol 了解多少

    理解和使用ES6中的Symbol

    每个 symbol 实例都是唯一的，如果比较两个 symbol 实例永远都是 false。

    不可枚举，就是不能用 Object.keys() 或者 for...in 访问到，所以在对象中，可以用 symbol 定义一些不需要对外操作和访问的属性。用 JSON.stringify() 的时候 Symbol 属性也会被排除。如果要获取的话可以通过特定的 API： Object.getOwnPropertySymbols(obj) 和 Reflect.ownKeys(obj)。

    使用 symbol 定义常量，定义私有化属性/方法。

    symbol.for() 会查找给定的 key，如果有就返回，没有就新建。

  • == 和 ===

    == 执行类型转换，=== 不类型转换

    ==

    1. 判断类型，不同 -> 转换类型 -> 严格比较
    2. 若两个都是对象，比较地址
    3. 如果一个是对象一个不是，则调用对象的 valueOf 或 toString 取得原始值再比较

    valueOf Object 对象方法，内置对象重写了该函数。默认返回对象本身

    toString 返回表示该对象的字符串，内置对象重写。默认返回 [object type]，所以可以通过 Object.prototype.toString.call(obj)（返回 object obj的类型）来检测 obj 的类型

    ===

    1. 判断类型，不同 -> false
    2. 判断值
    3. NaN 和任何值都不全等

    Object.is

    1. +0
    2. -0
    3. NaN
    • 有没有想过为什么会有两种判断方式

项目

  • 项目难点

    项目中有一个个人信息面板，是使用弹窗组件来完成的，就是点击个人中心然后出现一个弹窗，上面要展示一个二维码。这个二维码是通过一个第三方包生成的，用法是传入一个字符串和一个 canvas 对象，就会根据字符串生成一个二维码并且绘制到 canvas 上。然后当时的问题是这个二维码没有展示出来。

    那我的思路就是，字符串、生成、绘制这三个之中哪个出现了问题。然后我在排查过程中发现这个二维码一闪而过，就是绘制出来又被覆盖的感觉，我就考虑到应该是一个渲染顺序的问题。

    然后我在几个生命周期里面打印了 canvas 元素，首先点击父组件上的个人中心，然后父组件会执行 beforeUpdate，然后子组件 beforeUpdate、updated，父组件 updated，然后我发现 canvas 这个元素在子组件的 updated 阶段才能获取到，之前都是 null。

    我就去看了官方文档对生命周期的描述，DOM 重新渲染之后会调用 updated，这个时候可以进行依赖 DOM 的操作，但还是建议在计算属性或者 watch 里面修改。但 watch 里面还获取不到 canvas 标签，然后也是官网里面看到有 nextTick 这个 API，可以将回调延迟到下次 DOM 更新循环之后执行。我就把绘制二维码的代码放到 nextTick 的回调里面，问题就解决了。

    其实最开始我意识到渲染顺序的时候，是直接用 setTimeout 解决的，也可以完成，当时也好奇嘛，然后从这里深挖也了解到了事件循环机制，然后还发现 vue 这个 nextTick 源码里面是用 promise 实现的，如果浏览器不支持 promise 就会用 setTiemout 来实现，所以我最开始的解决方式也是误打误撞吧，觉得还挺神奇的。这件事情比较早了，就从这件事情上我意识到不能只用框架，还是要更深入的理解，像之前只是背背几个生命周期肯定是不行的。我当时还因为这个写了一个博客，印象还挺深刻的。
