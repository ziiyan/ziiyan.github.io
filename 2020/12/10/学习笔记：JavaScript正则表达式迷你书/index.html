学习笔记：JavaScript正则表达式迷你书

2020-12-10 JavaScript, 正则表达式 评论

正则表达式是匹配模式 ， 要么匹配字符，要么匹配位置 。

1. 正则表达式字符匹配攻略

1.1 两种模糊匹配

1.1.1 横向模糊匹配：长度不固定

  • {m, n} 连续出现最少 m 次，最多 n 次

    1.1.2 纵向模糊匹配：某一位字符有多种可能

  • [a, b, c] 表示该字符可以是 a/b/c 中的任意一个

1.2 字符组

1.2.1 范围

如果纵向模糊匹配里字符特别多，不可能一一枚举，则可以用范围表示法

  • [1-6a-fG-M] 等同于 [123456abcdefGHIJKLM]
    如果要匹配 -
  • [a\-z] [-az] [az-]

    1.2.2 排除

    可以是任何东西，除了 a/b/c
  • [^abc]

    1.2.3 常见简写

  • \d [0-9] 数字 digit
  • \D [^0-9] 非数字
  • \w [0-9a-zA-Z_] 数字、大小写字母、下划线。单词字符word
  • \W [^0-9a-zA-Z_] 非单词字符
  • \s [ \t\v\n\r\f] 空格、水平制表符、垂直制表符、换行符、回车符、换页符。空白符space
  • \S [^ \t\v\n\r\f] 非空白字符
  • . [^\n\r\u2028\u2029] 通配符，几乎任意字符。换行符、回车符、行分隔符、段分隔符除外。

1.3 量词：重复

1.3.1 简写

  • {m,} 至少出现 m 次
  • {m} 出现 m 次
  • ? {0, 1}
  • + {1,}
  • * {0,}

    1.3.2 贪婪匹配与惰性匹配

  • 贪婪：有多少匹配多少
  • 惰性：只要满足条件就停止
    默认贪婪，后方加上 ? 为惰性

1.4 分支

一个模式可以实现横向和纵向模糊匹配，而多选分支可以支持多个子模式任选其一

  • (p1|p2|p3) |管道分隔 p1/p2/p3 任选其一
    如果 p1 匹配，则不继续尝试

2. 正则表达式位置匹配攻略

2.1 什么是位置

相邻字符之间的位置

2.2 如何匹配位置

  • ^
  • $
  • \b
  • \B
  • (?=p)
  • (?!p)

2.2.1 ^ 和 $

  • ^（脱字符） 匹配开头，多行匹配中匹配每行开头
  • ¥（美元符号）匹配结尾，多行匹配中匹配每行结尾

    开头和结尾用 # 替换

    多行匹配模式（即有修饰符 m）时

2.2.2 \b 和 \B

  • \b 单词边界
    • 也就是 \w 和 \W 之间的位置，也包括 ^ 和 \w 之间的位置，和 \w 与 $ 之间的位置

\w 即 [0-9a-zA-Z_]，\W 是除此之外的任意字符

  1. [ 和 J 即 \W 和 \w
  2. S 和 ] 即 \w 和 \W
  3. 空格 和 L 即 \W 和 \w
  4. 1 和 . 即 \w 和 \W
  5. . 和 m 即 \W 和 \w
  6. 4 和 结尾 即 \w 和 $
  • \B 非单词边界
    • 与 \b 相反
    • 具体就是 \w 和 \w，\W 和 \W，^ 和 \W，\W 和 $

2.2.3 (?=p) 和 (?!p)

  • (?=p) 其中 p 是一个子模式，表示 p 前面的位置，或者说该位置后面的字符要匹配 p
    比如 (?=l) 表示 l 字符前面的位置
  • (?!p) 则相反
  • (?<=p) 表示 p 后面的位置
  • (?<!p) 则相反

2.3 位置的特性

可以将位置理解为空字符 ""
hello == "" + "h" + "" + "e" + "" + "l" + "" + "l" + "" + "o" + "";
hello == "" + "" + "hello";
所以 /^hello$/ 写成 /^^hello$$$/ 是可以的
即位置可以写成多个

例：长度 6-12 位，数字、小写字母、大写字母组成，至少 2 种字符

  1. 简化
  2. 必须包含数字
  3. 至少 2 种字符
  4. 理解 /(?=.*[0-9])^/
  5. 不能全是数字
